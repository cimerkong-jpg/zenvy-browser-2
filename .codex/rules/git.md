# Git Rules

## Before Editing
- Check worktree state before code changes when practical.
- Identify files already modified before touching them.
- Treat existing uncommitted changes as user-owned unless proven otherwise.

## During Work
- Do not revert user changes unless explicitly requested.
- Do not run destructive git commands without explicit approval.
- Do not stage, commit, amend, rebase, reset, or push unless requested.
- Keep changes scoped so the diff is reviewable.

## Reporting
- Report files changed.
- Mention relevant pre-existing changes when they affect the task.
- If a command changes generated files or lockfiles, explain why.
