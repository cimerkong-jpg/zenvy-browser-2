# Codebase Orientation Skill

Use when the user asks to understand a project, folder, module, feature, or unfamiliar code path.

## Use With
- `rules/context.md`
- `workflows/codebase-orientation.md`
- `roles/researcher.md`

## Procedure
1. Start from the user request, active file, entry point, or error.
2. Search for related symbols, routes, scripts, config, tests, and call sites.
3. Read only the files needed to map the behavior.
4. Separate confirmed facts from assumptions.
5. Return a concise map with key files and data flow.

## Output
- What area was inspected.
- Key files and responsibilities.
- Main data flow.
- Unknowns or recommended next inspection.
