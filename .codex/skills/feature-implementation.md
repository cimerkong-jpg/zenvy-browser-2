# Feature Implementation Skill

Use when adding or changing user-facing behavior.

## Use With
- `roles/implementer.md`
- `workflows/feature-delivery.md`
- `rules/scope.md`
- `rules/testing.md`

## Procedure
1. Clarify the user goal and acceptance criteria.
2. Find similar existing behavior and reuse its patterns.
3. Implement the smallest complete change.
4. Handle loading, empty, disabled, error, and success states when relevant.
5. Verify the primary flow and important edge cases.

## Output
- Behavior delivered.
- Files changed.
- Verification result.
- Known limitations or follow-up.
