# Test Verification Skill

Use when choosing checks, adding tests, or verifying a completed change.

## Use With
- `roles/tester.md`
- `rules/testing.md`
- `checklists/pre-final.md`

## Procedure
1. Identify the behavior that needs confidence.
2. Choose the smallest useful check: typecheck, unit, integration, E2E, build, lint, or manual check.
3. Add focused tests when the risk justifies it.
4. Run the relevant command when practical.
5. Report exact result and remaining gaps.

## Output
- Verification strategy.
- Tests or commands run.
- Result.
- Remaining gaps.
